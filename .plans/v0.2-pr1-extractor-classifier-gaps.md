## Context the engineer needs

### Why this matters

`catalog.json` (the primary deliverable, CC0-licensed, language-agnostic) is built by:
1. `scripts/extract_patterns.py` walks `nemweb-mirror/` and classifies every file into `(repo, retention_tier, intra_repo_id)`.
2. `scripts/merge_catalog.py` overlays `patterns/curated/*.yaml` onto the auto output.
3. The `write_json()` emitter groups the rows by `(repo, intra_repo_id)` and writes `patterns/auto/catalog.json`.

When the classifier can't identify a file, it dumps the row into one of these **placeholder** `intra_repo_id` values: `UNPARSED`, `UNKNOWN`, `ROOT_AUX`, `DOCUMENTATION_AUX`, `MTPASA_DATA_EXPORT`. The `write_json` step then collapses every file with the same placeholder into a single fake "dataset" entry, silently losing rows to row-collapse.

### Dead-code discovery

During plan authoring, the MMSDM bulk-zip, MONTH_ROOT_AUX, and SQLLOADER_AUX branches at `scripts/extract_patterns.py:275-287` were found to be **dead code**. The guards use `len(rel) == 1`, `len(rel) == 2`, and `len(rel) == 3`, but `rel` is derived from the full URL path **including the filename segment** — so for `MMSDM/2009/MMSDM_2009_07.zip`, `rel = ['2009', 'MMSDM_2009_07.zip']` (length 2, not 1).

Control-flow verification:

```python

# MMSDM/2009/MMSDM_2009_07.zip:           rel = ['2009', 'MMSDM_2009_07.zip']              len=2
# MMSDM/2009/MMSDM_2009_07/AUTORUN.INF:   rel = ['2009', 'MMSDM_2009_07', 'AUTORUN.INF']   len=3
# MMSDM/2009/MMSDM_2009_07/MMSDM_Historical_Data_SQLLoader/CTL/<file>:  rel len=5
```

### Aux-bucketing design decision (locked for this plan)

For the ~30 legit-aux rows (CD-autorun chrome like `AUTORUN.INF`, `Back.gif`, `nemlogo1.gif`, `shelexec.exe`, plus MMSDM docs like `README.txt`, `Readme.htm`, `Participant_Monthly_DVD.pdf`), the `intra_repo_id` is derived from the filename template by:

1. Stripping skeleton tokens (both `<dN>` form from `skeletonize()` and `{token}` form from `label_digit_positions()`).
2. Replacing non-alphanumeric runs with `_`.
3. Trimming leading/trailing `_`.
4. **Preserving source-byte casing** — no uppercase/lowercase folding.

Examples:

- `AUTORUN.INF` → `AUTORUN_INF`
- `Readme.htm` → `Readme_htm`
- `readme.htm` → `readme_htm` (DISTINCT from above — case variants stay distinct, per §3.1)
- `nemlogo<d1>.gif` → `nemlogo_gif`
- `nemlogo{d1}.gif` → `nemlogo_gif` (same result regardless of token form)
- `marketnoticedata_{yearmonth}.par` → `marketnoticedata_par`

The `retention_tier` carries the aux signal (`MONTH_ROOT_AUX`, `DOCUMENTATION_AUX`, `ROOT_AUX`, `MARKET_DATA_AUX`). Each aux filename template gets its own `(repo, intra_repo_id, retention_tier)` tuple → no collapse in `write_json`.

```python
# At module level, near the other helpers (before extract_mmsdm_table):

# MMSDM SQLLoader view files come in two filename dialects.
# Pre-2024 (~): PUBLIC_ARCHIVE#<TABLE>#FILE<NN>#<date>.<ext>  (URL-encoded '#' = %23)
# 2024-present: PUBLIC_DVD_<TABLE>_<yearmonth>.<ext>
# §3.1 byte-exact discipline: both dialects coexist for back-catalogue months;
# treat as two distinct patterns, not one. Case is preserved as-served.
_MMSDM_DVD_RE = re.compile(
    r"^PUBLIC_DVD_(?P<table>.+)_\d{6}\.(?:ctl|CTL|DATA|fmt|bcp|zip)$"
)


def extract_mmsdm_table(filename: str) -> str | None:
    """Extract the MMSDM table name from a SQLLoader-view filename.

    Supports two dialects observed in the mirror:
      * `PUBLIC_ARCHIVE#<TABLE>#FILE<NN>#<date>.<ext>`
        (URL-encoded as `PUBLIC_ARCHIVE%23…`; pre-2024 archive).
      * `PUBLIC_DVD_<TABLE>_<yearmonth>.<ext>`
        (2024-present; underscore-delimited).

    Returns None if neither dialect matches, so the caller can emit an
    UNPARSED placeholder and surface it as a classifier-gap signal.
    """
    decoded = urllib.parse.unquote(filename)
    if "#" in decoded:
        parts = decoded.split("#")
        if len(parts) >= 2 and parts[0] in ("PUBLIC_ARCHIVE", "PUBLIC"):
            return parts[1]
        if len(parts) >= 2:
            return parts[1]
        return None
    m = _MMSDM_DVD_RE.match(decoded)
    if m:
        return m.group("table")
    return None
```

```python
# Aux / CD-chrome / documentation boilerplate filenames are classified with a
# stem-derived intra_repo_id so each filename stays a distinct dataset entry
# rather than all collapsing into a shared AUX placeholder (which write_json
# would then fold into a single catalog row, losing 20+ rows silently).
#
# Byte-exact casing (§3.1) is preserved — do NOT uppercase/lowercase. Case
# variants like Readme.htm vs readme.htm are AEMO-served distinct files and
# must emit distinct aux_ids.
#
# The helper accepts a filename **template** — output of skeletonize() in
# <dN> angle-bracket form, or output of label_digit_positions() in {token}
# curly-brace form, or a bare filename. All three produce the same result
# when the filename is identical modulo digit-runs.
_AUX_ID_TOKEN = re.compile(r"<[^>]+>|\{[^}]+\}")
_AUX_ID_NONWORD = re.compile(r"[^A-Za-z0-9]+")


def aux_id_from_filename_template(tpl: str) -> str:
    """Derive a stable, case-preserving, stem-based intra_repo_id for an aux file.

    Strips skeleton tokens (both `<dN>` and `{token}` forms), replaces
    non-alphanumeric runs with `_`, and trims leading/trailing `_`. Source
    byte casing is preserved so two AEMO-served filenames that differ only
    in case get distinct ids.

    Examples::

        AUTORUN.INF          -> AUTORUN_INF
        Readme.htm           -> Readme_htm        (distinct from readme.htm)
        readme.htm           -> readme_htm
        nemlogo<d1>.gif      -> nemlogo_gif
        nemlogo{d1}.gif      -> nemlogo_gif
        marketnoticedata_{yearmonth}.par -> marketnoticedata_par
    """
    stripped = _AUX_ID_TOKEN.sub("", tpl)
    collapsed = _AUX_ID_NONWORD.sub("_", stripped)
    trimmed = collapsed.strip("_")
    return trimmed if trimmed else "AUX"
```
