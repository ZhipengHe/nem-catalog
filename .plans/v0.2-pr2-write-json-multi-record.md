**Goal:** Eliminate the `write_json` row-collapse bug. Curated YAML schema is unchanged — humans still write one tier dict; merge code lifts it into the array shape.

**Tech Stack:** Python 3 stdlib only for runtime SDK (`src/nem_catalog/`). Build pipeline (`scripts/`) keeps current deps (`jsonschema`, `pyyaml` via dev). Tests use `pytest` + `pytest-cov`. No new runtime deps — v0.2 stays stdlib-pure.

## Context the engineer needs

### Why this matters

`catalog.json` is built by:

1. `scripts/extract_patterns.py` walks `nemweb-mirror/` and emits flat `(repo, retention_tier, intra_repo_id, path_template, filename_template, …)` rows. `write_json()` then groups rows by `(repo, intra_repo_id)` and nests tier records under each dataset.
2. `scripts/merge_catalog.py` overlays `patterns/curated/*.yaml` onto the auto output and writes the published `catalog.json`.
3. SDK `src/nem_catalog/catalog.py` `Catalog.resolve(key, from_, to_)` reads `tiers` and expands each tier's path/filename templates against the date range.

The bug is at `scripts/extract_patterns.py:1083`:

```python
datasets[key]["tiers"][tier_name] = tier_record
```

When the flat CSV has multiple legitimately-distinct rows for the same `(repo, intra_repo_id, retention_tier)` triple, all but the last are silently overwritten. Measured against `reference/URL-CONVENTIONS.csv` state `d3b9cae` (2026-04-26):

- **474 colliding groups** (after AUX exclusion)
- **1033 rows silently dropped** from the published catalog

Every dropped row is a real dataset path/filename family the consumer cannot reach via `resolve()`.

### Sub-pattern breakdown (load-bearing for test fixtures)

| Sub-pattern | Groups | Rows lost | Shape | Example |
|---|---:|---:|---|---|
| `SAME_PATH_MULTI_FILE` | 448 | 700 | One path, N filename families | `Reports:VicGas [CURRENT]` = 1 path × 91 filenames; 13 `MMSDM:MMS_DATA_MODEL_v* [DOCUMENTATION]` (141 rows lost) |
| `MIXED` | 7 | 293 | Parent dir w/ both sub-streams + same-path families | `Reports:STTM [CURRENT]` (94 rows / 3 paths / 94 filenames) |
| `DUPLICATE_STRADDLE` | 14 | 14 | Parent + `/DUPLICATE/` sibling, same filename family | `Reports:Dispatch_SCADA [CURRENT]`, `Reports:Dispatch_IRSR`, etc. |
| `MULTI_SUBDIR` | 5 | 26 | N distinct path+filename pairs, no DUPLICATE | `Reports:GSH [ARCHIVE]` = 13 streams |

Reconcile: 448 + 7 + 14 + 5 = 474 groups; 700 + 293 + 14 + 26 = 1033 rows lost.

### Field-variance (drives the schema decision)

Across the 474 colliding groups:

- `path_template` varies in **5%** (26/474)
- `filename_template` / `filename_regex` vary in **97%** (460/474)
- `anomaly` varies in **0%** (never within a group)
- `cadence` is `_infer_cadence(repo, tier_name)` — uniform across records by construction
- `retention_hint_unverified_days` is set only by curated overlay (`patterns/curated/reports.yaml` line 7) on `Reports:DispatchIS_Reports [CURRENT]`, which is a single-record tier — **vacuously uniform across colliding groups today**
- `observed_range` derives from per-row `first_seen_snapshot` / `last_seen_snapshot` — **intrinsically per-record**
- `time_granularity` is inferred from `filename_template` — **per-record**

Conclusion: full per-record `$defs.Tier` (fields unchanged, container becomes array) is the cleanest shape. Tier-level metadata that's uniform-by-construction (`cadence`, `retention_hint_unverified_days`) is broadcast to all records via the extractor; the redundancy is harmless and predictable.

### Design lock (2026-04-26)

These decisions were locked after a recon pass against current data. Do not deviate without re-running the codex alignment audit.

**1. Schema shape — always-list, no union.**

```jsonc
// $defs.Dataset.tiers, before:
"additionalProperties": { "$ref": "#/$defs/Tier" }

// After:
"additionalProperties": {
  "type": "array",
  "minItems": 1,
  "items": { "$ref": "#/$defs/Tier" }
}
```

`$defs.Tier` itself unchanged. Single-record tiers become 1-element arrays. No `single | array` polymorphic union; consumers always iterate.

**2. `catalog.resolve()` semantics — per-record everywhere.**

| Aspect | Today | Post-fix |
|---|---|---|
| Token-resolvability skip | All-or-nothing per tier | Per-record; sibling records continue |
| `observed_range` filter | Tier-level | Per-record |
| Rolling-tier picker | `tiers[T].get("retention_hint_unverified_days")` | `tiers[T][0].get("retention_hint_unverified_days")` (uniform-by-construction) |
| `NonResolvableTemplateError` raise | Every selected tier non-temporal AND no URLs | Every selected **record** non-temporal AND no URLs |
| Skipped warning | `"skipped tier {T}"` | `"skipped tier {T} record {i} of {N}"` |

**3. Schema version bump: `1.0.0 → 2.0.0`.** Major; SDK rejects unknown major versions per existing semantics.

**4. Extractor invariant (regression-tested):** every non-AUX CSV row maps to exactly one emitted record under `tiers[T][i]`. No silent drops. Per-sub-pattern coverage tests required.

**5. Merge-layer adaptation:** `_merge_tiers` broadcasts curated fields to every record in the list; `_insert_curated_only` wraps the curated tier dict as `[tier_dict]`. Curated YAML files (`reports.yaml`, `mmsdm.yaml`, `nemde.yaml`, `fcas_causer_pays.yaml`) are **unchanged** — humans still write one tier dict per tier name.

### Out of scope (deferred)

- **Per-record `dataset_keys` enumeration.** Sub-keys per filename family (e.g. `Reports:VicGas/{filename-template}`) belong in `#15 list_urls()`, not here.
- **1.x ↔ 2.x compatibility shim.** No backward-compat layer in the SDK. Consumers pinning schema-major must update.
- **MMSDM table-rename merging.** Unchanged from current "out of scope" status.
- **#16 observed-range retention** (replace `retention_hint_unverified_days: int` with `{min, max, confidence}`). Likely slips to v0.2.1; if it lands in v0.2, batch its schema delta into the same 2.0.0 bump.

### Ground-truth discipline invariants (do NOT violate)

- **Byte-exact case-sensitivity.** §3.1 of `reference/NEMWEB-STRUCTURE.md` is load-bearing. No `.casefold()` / `.lower()` / `.upper()` anywhere that affects identity comparison or aux_id derivation.
- **Never hand-write a template or regex.** Grep `reference/URL-CONVENTIONS.csv` or the mirror for real examples first.
- **Do not modify `nemweb-mirror/`, `reference/`, `catalog.json`, `patterns/auto/`, or `uv.lock`** except via the normal regenerate commands. Pre-commit excludes these paths from formatters; the byte-preserved contract depends on it.
- **Fix the code, not the captures.** If a test fails against the mirror, add a `--refresh` flag. Never `rm -rf nemweb-mirror/` as a workaround.
- **Curated overlay boundary.** `patterns/curated/` is human judgment only (`anomaly_note`, `schema_source`, `query_shape`, `curated_only: true` placeholders). Do NOT add curated overrides to paper over fixes the merge or extractor should make.
