# nem-catalog v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `nem-catalog` v0.1 — a language-agnostic machine-readable URL catalog for AEMO NEMWEB, with a pure-stdlib Python SDK, published to PyPI and GitHub Pages, with a weekly auto-refresh GitHub Actions workflow.

**Architecture:** Hybrid catalog generation: `extract_patterns.py` emits auto-derived patterns from a mirrored snapshot; `patterns/curated/` YAML overlays add human judgment (anomaly notes, retention hints, schema source pointers). A merge script combines them into `catalog.json` validated against `catalog.schema.json`. The Python SDK's primary API is deterministic `load(path_or_url)`; `fetch_latest()` is an opt-in convenience wrapper.

**Tech Stack:** Python 3.11+ (pure stdlib for the SDK; dev deps: pytest, ruff, mypy, pyyaml, jsonschema). GitHub Actions for weekly cron + PyPI trusted publishing. GitHub Pages for static JSON hosting. `uv` as package manager. Conventional Commits.

### Problem Statement

AEMO publishes the Australian National Electricity Market's operational data as a large, heterogeneous, case-sensitive, partially-rolling file tree at `https://nemweb.com.au/`. The tree has four logical repositories (`Reports`, `MMSDM`, `NEMDE`, `FCAS_Causer_Pays`), each with different organizational conventions, temporal semantics, and naming grammars. There is no official machine-readable index of this tree. Every downstream Python tool that consumes NEMWEB (NEMOSIS, NEMSEER, NEMED, opennem/nemweb, nem-data, owebeeone/NEMWeb, TheSkorm/nemweb — at least 7+ active projects) reimplements URL-construction logic in its own codebase. Non-Python users (R, Julia, shell) are largely unserved.

`nem-catalog` fills this gap by publishing a versioned JSON catalog + JSON Schema that maps `(dataset, time_range) → list[candidate_url]` for all four repositories.

### Primary Persona (set during DX review)

Non-Python scripter (R, Julia, shell) doing ad-hoc NEMWEB data work. Secondary: Python tool-builder evaluating as a dependency. **Explicit non-persona:** Python researcher wanting a DataFrame — pointed to NEMOSIS in the README. Target TTHW for primary: Champion tier (<2 min via `curl | jq`).

### Premises (all post-review revisions applied)

1. **v0.1 is URL pattern resolution only.** Dataset identity = `(repo, intra_repo_id)`, matching existing extractor. Canonical keys (exact NEMWEB case):
   - `MMSDM:<TABLE>` — uppercase MMSDM table token (e.g., `MMSDM:DISPATCHPRICE`)
   - `Reports:<Stream>` — stream subdir name as it appears in URL path (e.g., `Reports:DispatchIS_Reports`)
   - `NEMDE:<FilePrefix>` — filename prefix before first digit run (e.g., `NEMDE:NemPriceSetter`)
   - `FCAS_Causer_Pays:CAUSER_PAYS` — single logical dataset, one file/year; NOT year-keyed
   No schema coverage beyond a single MMSDM tech-specs portal pointer.

2. **Canonical artifact = versioned JSON + published JSON Schema.** Python SDK is a thin reference (target 150–250 LOC, hard cap 300).
   **Tier routing:** for `Reports:*` with both CURRENT and ARCHIVE tiers, if `from_` is older than `catalog.as_of - retention_hint_unverified_days`, fan out ARCHIVE patterns; if `to_` is within retention window, fan out CURRENT; overlapping → both. Single-tier repos (MMSDM, NEMDE, FCAS_Causer_Pays) expand single pattern set. SDK emits structured warning when `from_` is older than retention window.

3. **Distribution:** GitHub Releases (versioned `catalog-<CalVer>.json`) + GitHub Pages (stable `catalog.json` URL) + PyPI (`nem-catalog` Python SDK). Weekly GitHub Actions cron re-crawls live NEMWEB, diffs against mirror, regenerates catalog, opens PR on changes.

4. **Licensing:** MIT for all code, CC0 for `catalog.json`.

5. **Tiered success criteria.** See "Success Criteria" below.

6. **This repo is catalog-only.** A future `nem-downloader` is out of scope here.

7. **Single repo for v0.1.** Two-repo vs monorepo decision deferred until downloader is actually being built.

8. **Catalog records URL patterns + observed range + retention hint, NOT enumerated URLs.** SDK returns candidate URLs from pattern expansion; reachability (HEAD 200) is caller's responsibility. Field semantics:
   - `catalog.as_of` — one top-level ISO timestamp per catalog (mirror-crawl completion time). Per-record `as_of` deferred.
   - `observed_range` — inclusive min/max of content-time token parsed from filenames across mirrored instances. Does NOT imply continuity/completeness. Gaps possible.
   - `retention_hint_unverified_days` — applies only to rolling tiers (`Reports/CURRENT`). Omitted for append-only repos. v0.1 single-snapshot observation; v0.2 replaces with confidence range.

```

### Curation Rule for dataset_keys

Include a key in `dataset_keys` iff:
- `datasets[key].resolvable == true`, AND
- `intra_repo_id` is NOT in: `{DOCUMENTATION_AUX, ROOT_AUX, MONTH_ROOT_AUX, SQLLOADER_AUX, MONTHLY_BULK, MTPASA_DATA_EXPORT, UNKNOWN}` (extractor's AUX/placeholder keys)
- Additionally, `intra_repo_id` is NOT a file extension like `.dll`, `.exe`, `.bat` (UTILITIES junk — rejected by simple heuristic).

`raw_keys` lists every dataset key the extractor emitted. The `datasets` map always contains every entry; both keyed lists index into the same records.

### SDK Public API

```python
# Primary (library-pure, deterministic, no implicit I/O):
catalog = nem_catalog.load("catalog.json")                                   # local path
catalog = nem_catalog.load("https://...github.io/.../catalog-2026.04.18.json")  # explicit URL

# Opt-in convenience (network + cache + fallback):
catalog = nem_catalog.fetch_latest(cache_dir="~/.cache/nem-catalog")           # default live
catalog = nem_catalog.fetch_latest(catalog_version="2026.04.18")               # pin for reproducibility

# Methods on Catalog object:
urls = catalog.resolve("Reports:DispatchIS_Reports", from_="2025-04-01", to_="2025-04-02")
n = catalog.count("Reports:DispatchIS_Reports", from_="2025-04-01", to_="2025-04-02")
keys = catalog.list_datasets(filter="Dispatch")                                # curated only
raw = catalog.list_datasets(filter=None, include_raw=True)                     # full 367

# Catalog attributes:
catalog.schema_version      # "1.0.0"
catalog.catalog_version     # "2026.04.18"
catalog.as_of               # datetime
```

### Error Type Hierarchy

```python
class NemCatalogError(Exception):
    """Base exception for nem_catalog."""

class IncompatibleCatalogError(NemCatalogError):
    """Catalog schema major version not supported by this SDK."""

class NemCatalogFetchError(NemCatalogError):
    """Cold fetch failed and no cache available."""

class UnresolvableDatasetError(NemCatalogError):
    """Dataset exists in catalog but resolvable=false (directory-level anomaly)."""
```

Plus stdlib-native: `KeyError` (unknown key, with `difflib.get_close_matches` suggestion in message); `ValueError` (malformed time range, `to_ < from_`, wrong type). Every error message string ends with `See: https://<pages-url>/docs/troubleshooting#<error-name>`.

**Data-boundary conditions** (not errors): time range entirely outside `observed_range` → `warnings.warn(...)` + return empty list.

### Merge Semantics (5 rules)

Merge unit is the dataset key `(repo, intra_repo_id)`. Rules applied in order:

1. **Field overlap** (same field in `auto/` and `curated/` for same key): `curated/` wins; build emits warning to PR showing both values.
2. **Curated-only field** (e.g., `retention_hint_unverified_days`, `schema_source`, `anomaly_note`): `curated/` contributes unconditionally, no warning.
3. **Auto-only field** (e.g., `filename_regex`, `observed_range`): pass through untouched.
4. **Orphan curated dataset key** (key in `curated/` but absent from `auto/`): warn on first occurrence, fail build on 2nd consecutive weekly run (protects against transient WAF/crawl failures).
5. **Auto-only dataset key**: flows through to catalog as-is.

"""Error types for nem_catalog.

Every error message includes a `See: <docs_url>` suffix so CLI/library users
can jump directly to the troubleshooting entry.
"""

from __future__ import annotations

_DOCS_BASE = "https://zhipenghe.github.io/nem-catalog/docs/troubleshooting"


def docs_url(anchor: str) -> str:
    """Build a docs URL for the given troubleshooting anchor."""
    return f"{_DOCS_BASE}#{anchor}"


class NemCatalogError(Exception):
    """Base class for all nem_catalog errors."""


class _WithDocsUrl(NemCatalogError):
    """Internal mixin that appends the docs URL to the message."""

    _anchor: str = "general"

    def __init__(self, message: str) -> None:
        full = f"{message}. See: {docs_url(self._anchor)}"
        super().__init__(full)


class IncompatibleCatalogError(_WithDocsUrl):
    """Catalog schema MAJOR version is not supported by this SDK.

    SDK 0.1.x supports schemas with major == 1. Catalogs with major == 2 or
    higher require an SDK upgrade.
    """

    _anchor = "incompatible-catalog"


class NemCatalogFetchError(_WithDocsUrl):
    """Live fetch failed and no usable cache is available.

    Raised by `fetch_latest()` on cold network failure. If a cache exists,
    `fetch_latest()` warns and serves the cached copy instead of raising.
    """

    _anchor = "fetch-failed"


class UnresolvableDatasetError(_WithDocsUrl):
    """The requested dataset exists in the catalog but is marked resolvable=false.

    Raised by `resolve()` when the key is present but the record describes a
    directory-level anomaly with no files (e.g., NEXT_DAY_OFFER_ENERGY)SPARSE).
    """

    _anchor = "unresolvable-dataset"
```

def write_json(
    rows: list[dict],
    out_path: "Path | str",
    catalog_version: str,
    as_of: str,
    source_mirror_commit: str,
) -> None:
    """Emit a JSON Schema v1.0.0 conformant catalog from extractor rows.

    Groups flat rows by (repo, intra_repo_id) and nests tier records under
    each dataset. Preserves exact case of intra_repo_id (do not normalize).
    Called after classify() + aggregate() produce the flat rows list.
    """
    import json as _json
    from pathlib import Path as _Path

    datasets: dict[str, dict] = {}
    all_raw_keys: list[str] = []

    for row in rows:
        repo = row["repo"]
        intra_repo_id = row["intra_repo_id"]
        key = f"{repo}:{intra_repo_id}"
        tier_name = row["retention_tier"]

        if key not in datasets:
            all_raw_keys.append(key)
            datasets[key] = {
                "repo": repo,
                "intra_repo_id": intra_repo_id,
                "resolvable": True,
                "tiers": {},
                "query_shape": None,
                "schema_source": None,
                "anomaly_note": _anomaly_note_from_flag(row.get("anomaly_flag", ""), row),
            }

        tier_record = {
            "path_template": row["path_template"],
            "filename_template": row.get("filename_template") or None,
            "filename_regex": row.get("filename_regex") or None,
            "example": row.get("sample_filename", ""),
            "cadence": _infer_cadence(repo, tier_name),
        }

        # Only include observed_range if we have any observed values
        first = row.get("first_seen_snapshot")
        last = row.get("last_seen_snapshot")
        if first and last:
            tier_record["observed_range"] = {"from": first, "to": last}
        else:
            tier_record["observed_range"] = None

        datasets[key]["tiers"][tier_name] = tier_record

    # Curate dataset_keys: resolvable=true AND not in AUX/placeholder keys
    dataset_keys = _curate_keys(list(datasets.keys()), datasets)

    payload = {
        "schema_version": "1.0.0",
        "catalog_version": catalog_version,
        "as_of": as_of,
        "source_mirror_commit": source_mirror_commit,
        "placeholders": _DEFAULT_PLACEHOLDERS,
        "dataset_keys": dataset_keys,
        "raw_keys": sorted(all_raw_keys),
        "datasets": datasets,
    }
    _Path(out_path).write_text(_json.dumps(payload, indent=2, ensure_ascii=False) + "\n")


def _anomaly_note_from_flag(flag: str, row: dict) -> str | None:
    """Translate extractor anomaly_flag into a human-readable anomaly_note."""
    if not flag:
        return None
    if flag == "aemo_url_typo":
        return (
            f"Directory-level anomaly: AEMO published this stream with a malformed name. "
            f"URL is correct as stated — do NOT URL-encode special characters. "
            f"Treat as resolvable only if filename_template is not null."
        )
    if flag == "casing_mismatch_vs_sibling_tier":
        return (
            "CURRENT/ARCHIVE tiers use different casing for this stream. "
            "Preserve exact case per tier when constructing URLs; do not normalize."
        )
    return f"Extractor flag: {flag}"


def _infer_cadence(repo: str, tier: str) -> str:
    """Best-effort cadence label based on repo and tier name."""
    if repo == "Reports" and tier == "CURRENT":
        return "5min"
    if repo == "Reports" and tier == "ARCHIVE":
        return "daily_rollup"
    if repo == "MMSDM":
        if tier in {"DATA", "P5MIN_ALL_DATA", "PREDISP_ALL_DATA"}:
            return "5min"
        return "monthly_bulk"
    if repo == "NEMDE":
        return "daily"
    if repo == "FCAS_Causer_Pays":
        return "annual"
    return "unknown"


_AUX_SUFFIXES = {"_AUX", "DOCUMENTATION_AUX", "ROOT_AUX", "MONTH_ROOT_AUX", "SQLLOADER_AUX"}
_AUX_EXACT = {"MMSDM_MONTHLY_BULK", "MTPASA_DATA_EXPORT", "UNKNOWN", "UNPARSED"}
_UTILITY_EXTENSIONS = {".dll", ".exe", ".bat", ".sh", ".cmd", ".tar", ".gz"}


def _curate_keys(all_keys: list[str], datasets: dict) -> list[str]:
    """Return the curated user-facing subset of dataset keys.

    Rule: resolvable=true AND intra_repo_id not AUX/placeholder AND not utility-file.
    """
    out = []
    for key in sorted(all_keys):
        ds = datasets[key]
        if not ds["resolvable"]:
            continue
        iid = ds["intra_repo_id"]
        if iid in _AUX_EXACT:
            continue
        if any(iid.endswith(suf) for suf in _AUX_SUFFIXES):
            continue
        if any(iid.lower().endswith(ext) for ext in _UTILITY_EXTENSIONS):
            continue
        out.append(key)
    return out


_DEFAULT_PLACEHOLDERS = {
    "date": {"format": "yyyymmdd", "example": "20250407", "regex": "\\d{8}"},
    "timestamp": {"format": "yyyymmddhhmm", "example": "202504070445", "regex": "\\d{12}"},
    "aemo_id": {"format": "16-digit AEMO identifier", "example": "0000000513144978", "regex": "\\d{16}"},
    "year": {"format": "yyyy", "example": "2025", "regex": "\\d{4}"},
    "month": {"format": "mm", "example": "04", "regex": "\\d{2}"},
    "yyyymm": {"format": "yyyymm", "example": "202504", "regex": "\\d{6}"},
    "yyyymmddHHMM": {"format": "yyyymmddHHMM", "example": "202504070445", "regex": "\\d{12}"},
    "nn": {"format": "zero-padded 2-digit", "example": "00", "regex": "\\d{2}"},
    "d1": {"format": "single digit", "example": "1", "regex": "\\d{1}"},
    "d2": {"format": "two digits", "example": "04", "regex": "\\d{2}"},
    "d11": {"format": "11 digits", "example": "20220604167", "regex": "\\d{11}"},
    "d13": {"format": "13 digits", "example": "2011110909700", "regex": "\\d{13}"},

# Curated Patterns

Human-authored overlays on top of the machine-generated `patterns/auto/` catalog.

## What goes here

**Net-new human judgment only**, not a re-encoding of extractor output:

- `anomaly_note` — human-readable explanation for edge cases the extractor flags but can't fully describe.
- `retention_hint_unverified_days` — derived from mirror observation, confirmed by a human.
- `schema_source` — URLs to authoritative AEMO schema documentation.
- `query_shape` — per-record user affordance (what inputs does THIS dataset take).
- Deprecation markers for datasets AEMO is retiring.

## What does NOT go here

- `path_template`, `filename_template`, `filename_regex`, `observed_range` — these are the extractor's job. If the extractor is wrong, fix the extractor.
- Regenerated pattern rows from URL-CONVENTIONS.md — that file is already a build artifact.

## Merge semantics

See `scripts/merge_catalog.py` and `docs/architecture.md`:

1. Field overlap with auto → curated wins, build warns with both values.
2. Curated-only field → accept unconditionally.
3. Auto-only field → pass through.
4. Orphan curated key (not in auto) → warn on first occurrence, fail on 2 consecutive weekly runs.
5. Auto-only key → flow through to catalog.

## File format

One YAML file per repo. Top-level keys are dataset identifiers in `Repo:intra_repo_id` form.
```

```python
"""Merge patterns/auto/catalog.json with patterns/curated/*.yaml overlays.

Merge semantics (see docs/architecture.md):

1. Field overlap (same field in auto + curated for same dataset key):
   curated wins; emit WARNING with both values.
2. Curated-only field: accept unconditionally, no warning.
3. Auto-only field: pass through untouched.
4. Orphan curated dataset key (key in curated but absent from auto):
   warn on first occurrence, fail on 2 consecutive weekly runs.
5. Auto-only dataset key: flows through to catalog.

Usage:
    python scripts/merge_catalog.py --auto patterns/auto/catalog.json \\
        --curated patterns/curated/ --out catalog.json [--prior-fail-count N]

--prior-fail-count is the count of previous consecutive orphan-key failures for
this curated key. The weekly workflow tracks this and passes it in.
"""

from __future__ import annotations

import argparse
import copy
import json
import sys
from pathlib import Path

import jsonschema
import yaml

REPO_ROOT = Path(__file__).resolve().parent.parent
SCHEMA_PATH = REPO_ROOT / "schemas" / "catalog.schema.json"


def load_auto(path: Path) -> dict:
    return json.loads(path.read_text())


def load_curated(curated_dir: Path) -> tuple[dict[str, dict], dict[str, dict]]:
    """Return (per_key_overlays, per_repo_defaults).

    per_repo_defaults[repo] is the '__default__' block from <repo>.yaml if present.
    """
    overlays: dict[str, dict] = {}
    defaults: dict[str, dict] = {}
    for yml in sorted(curated_dir.glob("*.yaml")):
        repo_name = yml.stem
        content = yaml.safe_load(yml.read_text()) or {}
        if "__default__" in content:
            defaults[_title_case_repo(repo_name)] = content.pop("__default__")
        for key, overlay in content.items():
            if key in overlays:
                raise SystemExit(f"duplicate curated key across YAML files: {key}")
            overlays[key] = overlay
    return overlays, defaults


def _title_case_repo(stem: str) -> str:
    """Map YAML filename to repo enum value."""
    mapping = {
        "reports": "Reports",
        "mmsdm": "MMSDM",
        "nemde": "NEMDE",
        "fcas_causer_pays": "FCAS_Causer_Pays",
    }
    return mapping.get(stem.lower(), stem)


def merge(
    auto: dict,
    overlays: dict[str, dict],
    defaults: dict[str, dict],
    prior_fail_count: int,
) -> dict:
    """Apply curated overlays to the auto catalog. Returns merged catalog."""
    merged = copy.deepcopy(auto)
    datasets = merged["datasets"]
    warnings: list[str] = []
    orphans: list[str] = []

    # Apply per-key overlays
    for key, overlay in overlays.items():
        if key not in datasets:
            orphans.append(key)
            continue
        ds = datasets[key]
        _merge_record(key, ds, overlay, warnings)

    # Apply per-repo defaults for schema_source (and anything else)
    for key, ds in datasets.items():
        repo = ds["repo"]
        default = defaults.get(repo, {})
        for field, value in default.items():
            if ds.get(field) is None:
                ds[field] = value

    # Report warnings on stderr
    for w in warnings:
        print(f"WARNING: field overlap: {w}", file=sys.stderr)

    # Orphan handling
    if orphans:
        for key in orphans:
            print(f"ORPHAN CURATED KEY: {key} (not in auto catalog)", file=sys.stderr)
        if prior_fail_count >= 1:
            print(
                f"FAIL: {len(orphans)} orphan curated key(s) for 2 consecutive runs.",
                file=sys.stderr,
            )
            raise SystemExit(1)

    return merged


def _merge_record(key: str, ds: dict, overlay: dict, warnings: list[str]) -> None:
    for field, curated_value in overlay.items():
        if field == "tiers":
            _merge_tiers(key, ds, curated_value, warnings)
            continue
        auto_value = ds.get(field)
        # field overlap: both non-None
        if auto_value is not None and auto_value != curated_value:
            warnings.append(f"{key}.{field}: auto={auto_value!r} vs curated={curated_value!r}")
        ds[field] = curated_value


def _merge_tiers(key: str, ds: dict, curated_tiers: dict, warnings: list[str]) -> None:
    for tier_name, tier_overlay in curated_tiers.items():
        if tier_name not in ds["tiers"]:
            warnings.append(f"{key}.tiers.{tier_name}: curated-only tier (auto has no such tier)")
            ds["tiers"][tier_name] = tier_overlay
            continue
        auto_tier = ds["tiers"][tier_name]
        for field, curated_value in tier_overlay.items():
            auto_value = auto_tier.get(field)
            if auto_value is not None and auto_value != curated_value:
                warnings.append(
                    f"{key}.tiers.{tier_name}.{field}: auto={auto_value!r} vs curated={curated_value!r}"
                )
            auto_tier[field] = curated_value


def validate(catalog: dict) -> None:
    schema = json.loads(SCHEMA_PATH.read_text())
    jsonschema.validate(instance=catalog, schema=schema)


def main() -> int:
    parser = argparse.ArgumentParser()
    parser.add_argument("--auto", required=True, type=Path)
    parser.add_argument("--curated", required=True, type=Path)
    parser.add_argument("--out", required=True, type=Path)
    parser.add_argument("--prior-fail-count", type=int, default=0,
                        help="consecutive orphan-key failures in prior runs")
    args = parser.parse_args()

    auto = load_auto(args.auto)
    overlays, defaults = load_curated(args.curated)
    merged = merge(auto, overlays, defaults, args.prior_fail_count)

    try:
        validate(merged)
    except jsonschema.ValidationError as e:
        print(f"FAIL: merged catalog does not validate: {e.message}", file=sys.stderr)
        return 1
    print("schema validation: OK")

    args.out.parent.mkdir(parents=True, exist_ok=True)
    args.out.write_text(json.dumps(merged, indent=2, ensure_ascii=False) + "\n")
    print(f"wrote {args.out}")
    return 0


if __name__ == "__main__":
    sys.exit(main())
```

"""Catalog loading: primary `load()` and opt-in `fetch_latest()`.

`load()` is pure: given a path or URL, returns a Catalog object. No cache,
no fallback, no implicit I/O beyond the single read. Tool-builders adopt this.

`fetch_latest()` wraps load() with ETag-cached fetch + last-good fallback,
for script users. It lives in Task 10; this file stubs it.
"""

from __future__ import annotations

import json
import urllib.request
from pathlib import Path

from nem_catalog.catalog import Catalog
from nem_catalog.errors import IncompatibleCatalogError

_SUPPORTED_MAJOR = 1


def load(path_or_url: str | Path) -> Catalog:
    """Load a catalog from a local path or HTTPS URL.

    Deterministic: reads once, parses, validates the schema major version,
    returns a Catalog. Raises FileNotFoundError / ValueError / IncompatibleCatalogError
    but does NOT do caching or fallback.
    """
    if isinstance(path_or_url, Path) or not str(path_or_url).startswith(("http://", "https://")):
        raw = Path(path_or_url).read_text()
    else:
        req = urllib.request.Request(
            str(path_or_url),
            headers={"User-Agent": "nem-catalog-sdk/0.1"},
        )
        with urllib.request.urlopen(req, timeout=30) as resp:  # noqa: S310
            raw = resp.read().decode("utf-8")

    try:
        data = json.loads(raw)
    except json.JSONDecodeError as e:
        raise ValueError(f"invalid JSON: {e}") from e

    _check_compat(data.get("schema_version", "0.0.0"))
    return Catalog(data)


def _check_compat(schema_version: str) -> None:
    try:
        major = int(schema_version.split(".", 1)[0])
    except (ValueError, IndexError) as e:
        raise IncompatibleCatalogError(f"invalid schema_version string {schema_version!r}") from e
    if major != _SUPPORTED_MAJOR:
        raise IncompatibleCatalogError(
            f"catalog schema version {schema_version} not supported by SDK 0.1 "
            f"(supports {_SUPPORTED_MAJOR}.x)"
        )

"""Catalog class and query methods.

The Catalog object is immutable. Queries are pure functions over the loaded data.
Design notes:
  * resolve() emits a WARNING and returns [] when (from_, to_) falls entirely
    outside observed_range for the tier. Raising would force every caller to
    try/except around data-boundary conditions.
  * resolve() raises UnresolvableDatasetError for records with resolvable=false
    (directory-level anomalies like NEXT_DAY_OFFER_ENERGY)SPARSE).
"""

from __future__ import annotations

import difflib
import warnings
from datetime import date, datetime, timedelta
from typing import Any

from nem_catalog.errors import UnresolvableDatasetError

_NEMWEB_BASE = "https://nemweb.com.au"


class Catalog:
    """A loaded nem-catalog."""

    def __init__(self, data: dict[str, Any]) -> None:
        self._data = data

    # -- metadata accessors -------------------------------------------------

    @property
    def schema_version(self) -> str:
        return self._data["schema_version"]

    @property
    def catalog_version(self) -> str:
        return self._data["catalog_version"]

    @property
    def as_of(self) -> datetime:
        return datetime.fromisoformat(self._data["as_of"].replace("Z", "+00:00"))

    @property
    def datasets(self) -> dict[str, dict[str, Any]]:
        return self._data["datasets"]

    def dataset_keys(self) -> list[str]:
        return list(self._data["dataset_keys"])

    def raw_keys(self) -> list[str]:
        return list(self._data["raw_keys"])

    # -- list_datasets ------------------------------------------------------

    def list_datasets(self, filter: str | None = None, *, include_raw: bool = False) -> list[str]:
        """Return dataset keys, optionally case-insensitive substring-filtered.

        By default returns the curated subset. Pass include_raw=True to get
        the full set including AUX / placeholder entries.
        """
        keys = self.raw_keys() if include_raw else self.dataset_keys()
        if filter is None:
            return keys
        needle = filter.lower()
        return [k for k in keys if needle in k.lower()]

    # -- resolve + count ----------------------------------------------------

    def resolve(
        self,
        key: str,
        from_: str,
        to_: str,
        *,
        view: str | None = None,
    ) -> list[str]:
        """Expand a dataset to candidate URLs covering the date range.

        Parameters
        ----------
        key : dataset identifier (e.g., "Reports:DispatchIS_Reports")
        from_ : ISO date (YYYY-MM-DD) or datetime string, inclusive
        to_ : ISO date (YYYY-MM-DD) or datetime string, inclusive
        view : optional tier filter (e.g., "DATA" for MMSDM) — narrows fan-out

        Returns
        -------
        list[str] : candidate URLs. Caller is responsible for reachability.

        Raises
        ------
        KeyError : key not in catalog (with difflib suggestion)
        ValueError : to_ < from_, malformed dates
        UnresolvableDatasetError : record has resolvable=false
        """
        record = self._get_record(key)
        if not record.get("resolvable", True):
            raise UnresolvableDatasetError(
                f"dataset {key!r} is resolvable=false (directory-level anomaly)"
            )

        dt_from = _parse_date(from_)
        dt_to = _parse_date(to_)
        if dt_to < dt_from:
            raise ValueError(f"to_ ({to_}) must be >= from_ ({from_})")

        urls: list[str] = []
        any_tier_produced_urls = False
        any_observed_overlap = False
        for tier_name, tier in record["tiers"].items():
            if view is not None and tier_name != view:
                continue
            observed = tier.get("observed_range")
            if observed and not _range_overlaps_observed(dt_from, dt_to, observed):
                continue
            if observed:
                any_observed_overlap = True
            tier_urls = _expand_tier(tier_name, tier, dt_from, dt_to)
            if tier_urls:
                any_tier_produced_urls = True
            urls.extend(tier_urls)

        if not any_observed_overlap and record["tiers"]:
            warnings.warn(
                f"resolve({key!r}, {from_!r}, {to_!r}): requested range is outside observed_range "
                f"for all tiers — returning empty list",
                stacklevel=2,
            )
            return []
        return urls

    def count(self, key: str, from_: str, to_: str, *, view: str | None = None) -> int:
        """Return the number of URLs resolve() would produce, without materializing."""
        return len(self.resolve(key, from_, to_, view=view))

    # -- internals ----------------------------------------------------------

    def _get_record(self, key: str) -> dict[str, Any]:
        if key not in self._data["datasets"]:
            close = difflib.get_close_matches(key, list(self._data["datasets"].keys()), n=3, cutoff=0.6)
            msg = f"unknown dataset key {key!r}"
            if close:
                msg += f". Did you mean: {', '.join(repr(c) for c in close)}?"
            raise KeyError(msg)
        return self._data["datasets"][key]


# ---- tier expansion ------------------------------------------------------


def _parse_date(s: str) -> datetime:
    """Parse an ISO date or datetime string. Accepts YYYY-MM-DD or YYYY-MM-DDTHH:MM:SSZ."""
    s = s.strip()
    if "T" in s:
        return datetime.fromisoformat(s.replace("Z", "+00:00"))
    return datetime.fromisoformat(s + "T00:00:00")


def _range_overlaps_observed(dt_from: datetime, dt_to: datetime, observed: dict) -> bool:
    obs_from = _parse_date_lenient(observed["from"])
    obs_to = _parse_date_lenient(observed["to"])
    return not (dt_to < obs_from or dt_from > obs_to)


def _parse_date_lenient(s: str) -> datetime:
    """Parse observed_range values, which may be year-only ('2011'), year-month ('2009-07'), or full ISO."""
    s = s.strip()
    if len(s) == 4:  # year
        return datetime.fromisoformat(f"{s}-01-01T00:00:00")
    if len(s) == 7:  # YYYY-MM
        return datetime.fromisoformat(f"{s}-01T00:00:00")
    if "T" not in s:
        return datetime.fromisoformat(s + "T00:00:00")
    return datetime.fromisoformat(s.replace("Z", "+00:00"))


def _expand_tier(tier_name: str, tier: dict, dt_from: datetime, dt_to: datetime) -> list[str]:
    """Expand a single tier's template across the requested date range."""
    filename_template = tier.get("filename_template")
    if not filename_template:
        return []  # directory-level anomaly with no files
    path_template = tier["path_template"]
    granularity = tier.get("time_granularity") or _infer_granularity_from_template(filename_template)

    urls: list[str] = []
    for dt in _iterate_dates(dt_from, dt_to, granularity):
        vars_ = _placeholder_values(dt, granularity)
        try:
            path = _format_template(path_template, vars_)
            fname = _format_template(filename_template, vars_)
        except KeyError:
            # Template uses placeholders we can't fill from dt alone (e.g., aemo_id).
            # Emit a template-resolved URL with unknown placeholders left in braces.
            path = _format_template(path_template, vars_, partial=True)
            fname = _format_template(filename_template, vars_, partial=True)
        urls.append(f"{_NEMWEB_BASE}{path}{fname}")
    return urls


def _infer_granularity_from_template(template: str) -> str:
    # Best-effort: pick the finest granularity placeholder present.
    for g in ("yyyymmddHHMM", "timestamp", "yyyymmdd", "date", "yyyymm", "month", "year", "yyyy"):
        if "{" + g + "}" in template:
            return g
    return "yyyymmdd"


def _iterate_dates(dt_from: datetime, dt_to: datetime, granularity: str):
    if granularity in {"yyyy", "year"}:
        for y in range(dt_from.year, dt_to.year + 1):
            yield datetime(y, 1, 1)
    elif granularity in {"yyyymm", "month"}:
        y, m = dt_from.year, dt_from.month
        while (y, m) <= (dt_to.year, dt_to.month):
            yield datetime(y, m, 1)
            if m == 12:
                y, m = y + 1, 1
            else:
                m += 1
    elif granularity in {"yyyymmddHHMM", "timestamp"}:
        # 5-minute granularity
        cur = dt_from
        while cur <= dt_to:
            yield cur
            cur += timedelta(minutes=5)
    else:  # yyyymmdd / date / fallback
        cur = dt_from
        while cur <= dt_to:
            yield cur
            cur += timedelta(days=1)


def _placeholder_values(dt: datetime, granularity: str) -> dict[str, str]:
    return {
        "date": dt.strftime("%Y%m%d"),
        "timestamp": dt.strftime("%Y%m%d%H%M"),
        "yyyymmdd": dt.strftime("%Y%m%d"),
        "yyyymmddHHMM": dt.strftime("%Y%m%d%H%M"),
        "yyyymm": dt.strftime("%Y%m"),
        "year": dt.strftime("%Y"),
        "yyyy": dt.strftime("%Y"),
        "month": dt.strftime("%m"),
        # Placeholders we can't fill from dt alone:
        # aemo_id, nn, d1, d2, d11, d13 — left empty.
    }


def _format_template(template: str, vars_: dict[str, str], *, partial: bool = False) -> str:
    """Substitute {placeholder} tokens. If partial=True, unknown placeholders are kept as '{name}'."""
    out = template
    for k, v in vars_.items():
        out = out.replace("{" + k + "}", v)
    return out
```

"""Catalog loading: primary `load()` and opt-in `fetch_latest()`.

load() — pure, deterministic, no implicit I/O beyond the one read.
fetch_latest() — opt-in convenience: live fetch, ETag cache, last-good fallback.
"""

from __future__ import annotations

import json
import os
import urllib.error
import urllib.request
import warnings
from pathlib import Path

from nem_catalog.catalog import Catalog
from nem_catalog.errors import IncompatibleCatalogError, NemCatalogFetchError

_SUPPORTED_MAJOR = 1
_DEFAULT_URL = "https://zhipenghe.github.io/nem-catalog/catalog.json"
_DEFAULT_RELEASE_URL_TEMPLATE = (
    "https://github.com/ZhipengHe/nem-catalog/releases/download/"
    "v{version}/catalog-{version}.json"
)
_USER_AGENT = "nem-catalog-sdk/0.1"


def load(path_or_url: str | Path) -> Catalog:
    """Load a catalog from a local path or HTTPS URL. Pure: no cache, no fallback."""
    if isinstance(path_or_url, Path) or not str(path_or_url).startswith(("http://", "https://")):
        raw = Path(path_or_url).read_text()
    else:
        req = urllib.request.Request(str(path_or_url), headers={"User-Agent": _USER_AGENT})
        with urllib.request.urlopen(req, timeout=30) as resp:  # noqa: S310
            raw = resp.read().decode("utf-8")

    try:
        data = json.loads(raw)
    except json.JSONDecodeError as e:
        raise ValueError(f"invalid JSON: {e}") from e

    _check_compat(data.get("schema_version", "0.0.0"))
    return Catalog(data)


def fetch_latest(
    cache_dir: str | Path | None = None,
    catalog_version: str | None = None,
) -> Catalog:
    """Live fetch + ETag cache + last-good fallback. For scripts and notebooks.

    Parameters
    ----------
    cache_dir : directory for cached catalog + ETag. Defaults to ~/.cache/nem-catalog.
    catalog_version : optional CalVer pin. When set, fetches the versioned release
        asset URL instead of the stable `latest` URL. Use for reproducible research.
    """
    cache_dir = Path(cache_dir or "~/.cache/nem-catalog").expanduser()
    cache_dir.mkdir(parents=True, exist_ok=True)

    if catalog_version:
        template = os.environ.get("NEM_CATALOG_RELEASE_URL_TEMPLATE", _DEFAULT_RELEASE_URL_TEMPLATE)
        url = template.format(version=catalog_version)
        cache_name = f"catalog-{catalog_version}.json"
    else:
        url = os.environ.get("NEM_CATALOG_URL", _DEFAULT_URL)
        cache_name = "catalog.json"

    cached_path = cache_dir / cache_name
    etag_path = cache_dir / f"{cache_name}.etag"
    cached_etag = etag_path.read_text().strip() if etag_path.exists() else None

    try:
        headers = {"User-Agent": _USER_AGENT}
        if cached_etag:
            headers["If-None-Match"] = cached_etag
        req = urllib.request.Request(url, headers=headers)
        with urllib.request.urlopen(req, timeout=30) as resp:  # noqa: S310
            body = resp.read()
            etag = resp.headers.get("ETag") or ""
        cached_path.write_bytes(body)
        if etag:
            etag_path.write_text(etag)
    except urllib.error.HTTPError as e:
        if e.code == 304 and cached_path.exists():
            # Revalidation hit — serve from cache
            pass
        else:
            return _serve_cache_with_warning(cached_path, str(e))
    except (urllib.error.URLError, TimeoutError) as e:
        return _serve_cache_with_warning(cached_path, str(e))

    try:
        data = json.loads(cached_path.read_text())
    except (json.JSONDecodeError, FileNotFoundError) as e:
        raise NemCatalogFetchError(f"catalog cache corrupt or missing: {e}") from e

    _check_compat(data.get("schema_version", "0.0.0"))
    return Catalog(data)


def _serve_cache_with_warning(cached_path: Path, error_detail: str) -> Catalog:
    if not cached_path.exists():
        raise NemCatalogFetchError(
            f"live fetch failed and no cache available: {error_detail}"
        )
    warnings.warn(
        f"nem_catalog: network error, serving cached catalog ({error_detail})",
        stacklevel=2,
    )
    data = json.loads(cached_path.read_text())
    _check_compat(data.get("schema_version", "0.0.0"))
    return Catalog(data)


def _check_compat(schema_version: str) -> None:
    try:
        major = int(schema_version.split(".", 1)[0])
    except (ValueError, IndexError) as e:
        raise IncompatibleCatalogError(f"invalid schema_version {schema_version!r}") from e
    if major != _SUPPORTED_MAJOR:
        raise IncompatibleCatalogError(
            f"catalog schema version {schema_version} not supported by SDK 0.1 "
            f"(supports {_SUPPORTED_MAJOR}.x)"
        )
```

# AEMO crawl coordination

`nem-catalog` mirrors public NEMWEB directory listings weekly for the express purpose of deriving URL patterns for research and tool-building use. This document explains what the crawler does, how to contact us, and how to request adjustments.

## What the crawler does

- Fetches **directory listings only** (HTML `index.html` pages), not data files.
- Throttles to **1 request per second globally** across all threads (see `scripts/nemweb_download.py::MIN_DELAY_S`).
- Uses `--gaps` incremental mode weekly and a full recrawl monthly as a cross-check.
- User-Agent: `nem-catalog-survey (+https://github.com/ZhipengHe/nem-catalog; directory-listing downloads only)`.
- Source IP: GitHub Actions hosted-runner IP range (variable; published at https://api.github.com/meta).

## What the crawler does NOT do

- No data file downloads (no zips, no CSVs, no XML).
- No repeated pathological polling.
- No authentication or credentialed access.

To request a rate-limit adjustment, allowlist, or ask the crawler to stop:
- Open a GitHub issue at https://github.com/ZhipengHe/nem-catalog/issues
- Tag it `aemo-coordination`.
- The repo maintainer (Zhipeng He) will pause the weekly workflow within 48 hours if requested.

## If we are blocked by the WAF

# Troubleshooting

## #unresolvable-dataset

`UnresolvableDatasetError` means the dataset key exists in the catalog but the record has `resolvable: false` — typically a directory-level anomaly (e.g., `NEXT_DAY_OFFER_ENERGY)SPARSE`) where AEMO published a directory with zero files.

**Fix:** check the `anomaly_note` field on the record for details. If you believe it should be resolvable, open an issue with a concrete example URL you expected.

## #incompatible-catalog

`IncompatibleCatalogError` means the catalog's `schema_version` MAJOR is not supported by your SDK version. SDK 0.1.x supports schema `1.x`.

**Fix:** upgrade the SDK (`pip install -U nem-catalog`) or pin an older catalog version.

## #fetch-failed

`NemCatalogFetchError` means the live fetch failed AND no cache was available.

**Fix:** check network connectivity; manually download the catalog from the Releases page and pass it to `nem_catalog.load()` directly.

## Empty result from resolve() with a warning

The requested date range is entirely outside the dataset's `observed_range`. No error — empty list + `warnings.warn`. Check the catalog record's `observed_range` for each tier; pick a range that overlaps.

## Key typo — KeyError suggestion

`KeyError` includes a `Did you mean` suggestion via `difflib`. Use the suggested spelling or browse `catalog.dataset_keys()`.

## curl + jq returns `null`

Your dataset key is wrong. List valid keys:

```bash
curl -s https://zhipenghe.github.io/nem-catalog/catalog.json \
  | jq '.dataset_keys[]' | grep -i YOUR_SEARCH
```

Write `.github/workflows/weekly-refresh.yml`:

```yaml
name: weekly-refresh

on:
  schedule:
    - cron: "0 3 * * 1"   # Mondays 03:00 UTC
  workflow_dispatch:       # manual trigger

permissions:
  contents: write
  pull-requests: write
  issues: write

jobs:
  refresh:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v5
      - name: Install uv
        uses: astral-sh/setup-uv@v7
      - name: Install project
        run: uv sync --extra dev

      - name: Determine crawl mode
        id: mode
        run: |
          DAY=$(date +%u)
          DATE=$(date +%d)
          if [ "$DATE" = "01" ] || [ "$DATE" = "02" ] || [ "$DATE" = "03" ] || [ "$DATE" = "04" ] || [ "$DATE" = "05" ] || [ "$DATE" = "06" ] || [ "$DATE" = "07" ]; then
            echo "mode=full" >> $GITHUB_OUTPUT
          else
            echo "mode=gaps" >> $GITHUB_OUTPUT
          fi

      - name: Crawl NEMWEB (incremental or full)
        id: crawl
        run: |
          set -e
          if [ "${{ steps.mode.outputs.mode }}" = "full" ]; then
            uv run python scripts/nemweb_download.py 10000 2>&1 | tee nemweb-mirror/download.log
          else
            uv run python scripts/nemweb_download.py --gaps 10000 2>&1 | tee nemweb-mirror/download.log
          fi
          # Count non-200 responses
          FAILED=$(grep -c "status=" nemweb-mirror/download.log || true)
          TOTAL=$(grep -c "fetch\[" nemweb-mirror/download.log || true)
          echo "failed=$FAILED" >> $GITHUB_OUTPUT
          echo "total=$TOTAL" >> $GITHUB_OUTPUT

      - name: Check WAF / block indicators (>5% non-200)
        id: block_check
        run: |
          FAILED=${{ steps.crawl.outputs.failed }}
          TOTAL=${{ steps.crawl.outputs.total }}
          if [ "$TOTAL" -gt 0 ]; then
            PCT=$(( 100 * FAILED / TOTAL ))
            echo "Failure rate: $PCT%"
            if [ "$PCT" -gt 5 ]; then
              echo "blocked=true" >> $GITHUB_OUTPUT
            fi
          fi

      - name: Open GH issue if blocked
        if: steps.block_check.outputs.blocked == 'true'
        env:
          GH_TOKEN: ${{ github.token }}
        run: |
          gh issue create \
            --title "crawl blocked: $(date -u +%Y-%m-%dT%H:%MZ) — ${{ steps.crawl.outputs.failed }}/${{ steps.crawl.outputs.total }} non-200" \
            --body "The weekly crawl encountered a high rate of non-200 responses. See workflow run for details. Catalog continues serving last-good artifact until this is resolved." \
            --label "aemo-coordination,bug"
          exit 1

      - name: Regenerate auto patterns
        run: uv run python scripts/extract_patterns.py

      - name: Track orphan-key prior fail count
        id: orphans
        run: |
          if [ -f .github/orphan-fail-count.txt ]; then
            PRIOR=$(cat .github/orphan-fail-count.txt)
          else
            PRIOR=0
          fi
          echo "prior=$PRIOR" >> $GITHUB_OUTPUT

      - name: Merge catalog
        id: merge
        continue-on-error: true
        run: |
          uv run python scripts/merge_catalog.py \
            --auto patterns/auto/catalog.json \
            --curated patterns/curated/ \
            --out catalog.json \
            --prior-fail-count ${{ steps.orphans.outputs.prior }}

      - name: Update orphan-key fail counter
        run: |
          if [ "${{ steps.merge.outcome }}" = "failure" ]; then
            echo $(( ${{ steps.orphans.outputs.prior }} + 1 )) > .github/orphan-fail-count.txt
            exit 1
          else
            echo 0 > .github/orphan-fail-count.txt
          fi

      - name: Detect diff
        id: diff
        run: |
          if git diff --quiet nemweb-mirror/ patterns/auto/ catalog.json; then
            echo "changed=false" >> $GITHUB_OUTPUT
          else
            echo "changed=true" >> $GITHUB_OUTPUT
          fi

      - name: Create PR on changes
        if: steps.diff.outputs.changed == 'true'
        uses: peter-evans/create-pull-request@v7
        with:
          commit-message: "chore(data): weekly catalog refresh"
          title: "chore(data): weekly catalog refresh ($(date -u +%Y-%m-%d))"
          body: |
            Weekly auto-refresh from `weekly-refresh.yml`.

            ### Change summary

            See the files tab for diffs in `nemweb-mirror/`, `patterns/auto/`, and `catalog.json`.

            ### Reviewer checklist

            - [ ] Any new datasets that need a `patterns/curated/` overlay?
            - [ ] Any anomaly warnings worth human judgment?
            - [ ] Any dropped datasets — deprecation note needed?
            - [ ] Merge warnings from the workflow logs resolved?
          branch: weekly-refresh-${{ github.run_id }}
          labels: weekly-refresh,data-catalog
```
